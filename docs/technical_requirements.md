# ⚙️ Technical Requirements

## System Requirements

| Requirement | Minimum |
|---|---|
| Python | 3.11+ |
| RAM | 4GB |
| Disk Space | 500MB |
| OS | Windows / macOS / Linux |

## Dependencies

| Package | Version | Purpose |
|---|---|---|
| flask | 3.1.3 | Web application framework |
| numpy | 2.3.0 | Numerical computations |
| pandas | 2.3.0 | Data manipulation |
| scikit-learn | 1.7.0 | ML models and pipelines |
| pytest | 9.0.2 | Unit testing |

Install all dependencies with:
```powershell
pip install -r requirements.txt
```

---

## Project Setup

### Step 1 — Clone
```powershell
git clone https://github.com/zzysyed/sales-forecasting.git
cd sales-forecasting
```

### Step 2 — Install dependencies
```powershell
pip install -r requirements.txt
```

### Step 3 — Generate data
```powershell
python data/generate_data.py
```

### Step 4 — Preprocess
```powershell
python -m src.data_preprocessing
```

### Step 5 — Train models
```powershell
python -m src.model_training
```

### Step 6 — Launch web app
```powershell
python -m src.web_app
```

### Step 7 — Run tests
```powershell
python -m pytest tests/ -v
```

## Troubleshooting

**`FileNotFoundError: Raw data not found`**
→ Run `python data/generate_data.py` first.

**`FileNotFoundError: No trained model found`**
→ Run `python -m src.model_training` first.

**`ModuleNotFoundError: No module named 'src'`**
→ Make sure you're running commands from the `sales-forecasting/` root folder using `python -m` syntax.

**Flask app not loading**
→ Check that port 5000 is not already in use. If it is, run:
```powershell
python -m src.web_app --port 5001
```
